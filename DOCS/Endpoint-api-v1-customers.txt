Endpoint: /api/v1/customers
Ruta:

Route::resource('customers', CustomersController::class);

Descripción:
Este endpoint gestiona los clientes disponibles en el sistema.

Parámetros de Entrada:
•  limit: Se puede ingresar un número entero o enviar la palabra "all" para retornar todos los registros (opcional).

•  search: Recibe tokens o palabras y busca por el campo nombre, teléfono, email, y nombre de sesión (opcional).

•  contact_name: Recibe tokens o palabras y busca por el nombre de contactos del cliente (opcional).

•  display_name: Recibe tokens o palabras y busca por el nombre principal (opcional).

•  phone: Recibe tokens o palabras y busca por el teléfono (opcional).

•  status_payment: Recibe dos parámetros: "prepaid" o "postpaid" (opcional).

•  orderByField: Recibe el campo por el cual se quiere ordenar los registros (opcional). Por defecto, ordena por la fecha de creación.

•  orderBy: Indica si quiere ordenar de forma ascendente (asc) o descendente (desc). Por defecto es ascendente (opcional).

Ejemplo de Solicitud:

GET /api/v1/customers?limit=10&search=John&orderByField=name&orderBy=desc HTTP/1.1
Host: yourapi.com
Authorization: Bearer <token>
